## Core Philosophy

The AKP specification is built on five principles:

<CardGroup cols={2}>
  <Card title="Wire-First" icon="cable-car">
    Every primitive is defined at the wire level first. Implementations in TypeScript, Python, Go, or Rust are secondary artifacts.
  </Card>
  <Card title="Composable" icon="puzzle-piece">
    Primitives compose without coupling. A Skill can use RAG without importing RAG internals вҖ” only the wire interface matters.
  </Card>
  <Card title="Versioned" icon="tag">
    All endpoints are versioned (`/v1/...`). Breaking changes require a major version bump. Minor changes are backward-compatible.
  </Card>
  <Card title="Observable" icon="chart-line">
    Every primitive emits OpenTelemetry spans. Cost, latency, and token usage are first-class observability signals.
  </Card>
</CardGroup>

## Type System

AKP defines a shared TypeScript type system exported from `@avant-dev/akp-core`. All wire types are derived from these TypeScript interfaces via JSON Schema generation.

```typescript
// Core type exports from @avant-dev/akp-core
export type {
  // Provider
  ProviderRequest,
  ProviderResponse,
  ProviderStreamChunk,
  ProviderConfig,
  ModelCapabilities,

  // Memory
  MemoryEntry,
  MemoryTier,
  MemoryWriteOptions,
  MemoryReadOptions,
  MemorySearchOptions,

  // RAG
  RagDocument,
  RagChunk,
  RagEmbedding,
  RagSearchResult,
  RagPipelineConfig,

  // Skills
  SkillDefinition,
  SkillInput,
  SkillOutput,
  SkillExecutionContext,
  SkillStreamChunk,

  // Kernel
  KernelConfig,
  KernelRequest,
  KernelResponse,
} from "@avant-dev/akp-core";
```

## Versioning Policy

AKP uses **semantic versioning** for both the specification and the reference SDK.

- **Major** (`v1 вҶ’ v2`): Breaking wire format changes. Old endpoints remain available for one release cycle.
- **Minor** (`v0.1 вҶ’ v0.2`): New primitives or endpoints added, backward-compatible.
- **Patch** (`v0.1.0 вҶ’ v0.1.1`): Bug fixes, clarifications, no behavioral changes.

The wire version is embedded in the URL path: `/v1/provider/complete`, `/v1/memory/read`, etc.
